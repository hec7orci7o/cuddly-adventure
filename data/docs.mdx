## Inicio rápido

1. **Iniciar sesión** en **GitHub**.
2. Establecer una configuración valida.
3. Escribir nuestro codigo ASM.

### Ejemplo de configuración

Especificación de lo que se pretende hacer
```
mov K, rb         ; SignExt(K) → BR(rb)
add ra,rb,rd      ; BR(ra) + BR(rb) → BR(rd)
ld (ra), rb       ; MEM(BR(ra)15-0) → BR(rb)
st rb, (ra)       ; BR(rb) → MEM(BR(ra)15-0)
beq ra, rb, K     ; si BR(ra)==BR(rb) salta a @ K
nop               ; no realiza ninguna acción
```

### Tipos de output
#### Binario
Cada línea esta compuesta por N número de bits, tantos como se especifico en la configuración.<br/>
Los bits en **rojo** simbolizan bits vacios y los **negros**, aquellos que están en uso.
El formato del vector de bits es de N-1..0, es decir si tenemos 32 bits, el bit 32 será el que 
este más a la izquierda situado en la posicion 31 y por otro lado, el 0 será el del extremo opuesto.
```
[31, 30, 29, ..., 2, 1, 0]	
```

#### Hexadecimal
La salida en Hexadecimal es la traducción secuencial de las líneas binarias.<br/>
Por defecto, la salida será con formato logisim para poderla importar y simular sin ningun tipo de esfuerzo.

## Configuration
Cada línea de configuración sigue este formato:
```
mov #K rb; 010 31:29 18:3 23:19;
<op> <cte | reg>[0..N]; <co> <pos_co> <pos_arg>[0..N]
```
